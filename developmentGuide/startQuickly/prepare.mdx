### 1. 申请 merchantId 和密钥

Powercard 与商户间的服务通信使用 AES + RSA 的方式对请求和响应的内容进行加解密，因此在正式接入前，需要先提前申请对应的
merchantId 和密钥。

**详情请参阅：**

- [通信加密](/developmentGuide/startQuickly/encryption)

### 2. 选择卡产品

Powercard 支持数字卡、虚拟卡、实体卡等不同的卡产品，不同的卡产品费率和使用场景不同，可以根据自己的业务需求选择一种或多种卡产品进行接入。

| 卡产品     | 支持卡类型   | 状态   | 是否需要KYC |
|---------|---------|------|---------|
| 万事达虚拟卡  | 虚拟卡     | 已上线  | 不需要     |
| VISA经典卡 | 虚拟卡/实体卡 | 已上线  | 需要      |
| VISA实体卡 | 实体卡     | 开发完成 | 需要      |

### 3. IP白名单

商户需提供服务器的固定 IP 地址，用于添加到 IP 白名单中。只有在 IP 白名单中的地址才能访问 Powercard 的服务，确保系统安全性。

### 4. 回调接口

商户需提供[webhook回调接口](/developmentGuide/startQuickly/callback)，回调 api 用于在特定事件发生时，PowerCard
向商户系统发送通知，告知操作结果或状态变化。商户需按规则实现回调接口，确保及时接受并处理通知。主要用于以下场景：

- 创建卡片
- 卡片状态变化
- 卡片充值
- 卡片消费
- 卡片注销

**详情请参阅：**

- [webhook回调](/developmentGuide/startQuickly/callback)
